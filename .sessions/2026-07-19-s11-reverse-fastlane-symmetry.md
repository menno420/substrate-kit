## What shipped

The R8 runtime advisory `check_fastlane_symmetry` cross-checks the two host fast-lane surfaces — `.github/workflows/auto-merge-enabler.yml` ARMS the head-branch prefixes that ride the auto-merge fast lane; the ci.yml claims-only guard (`case "$head_ref" in … claude/*) …`) CARDS the ones that must carry a session card. It covered only ONE direction:

- **Forward (R8) — `carded − armed`:** a prefix the guard cards but the enabler never arms → a benign guard that cards a branch the fast lane never touches.

The **dangerous** reverse direction was uncheckable at runtime:

- **Reverse (S11) — `armed − carded`:** a prefix the enabler arms (so it rides the fast lane) that the guard never cards merges **card-less** — the #451-race class. The catch: a card-less rider can be *intentional* (`claim/*` rides the lane card-free by design) or *accidental* (a new seat prefix armed but never carded). The advisory can't tell them apart from surface shape alone.

S11 closes it by giving ci.yml a **host-local ground truth**:

- `.github/workflows/ci.yml` — a `# fastlane-cardless: claim/` self-declaration comment beside the claims-only guard. Needs no dependency on the kit's `guards.FASTLANE_PREFIX_REGISTRY` (which an adopter never runs) — the host declares its own intentional card-less prefixes.
- `src/engine/checks/check_fastlane_symmetry.py` — `_declared_cardless_prefixes(text)` parses every `# fastlane-cardless: <prefix> …` line (whitespace/comma-separated tokens); the function now accumulates one `Finding` per fired leg. New reverse leg: `reverse_stray = armed − carded − declared_cardless` → `Finding(kind="fastlane-cardless-drift")` naming the accidental card-less riders. **Self-gated on the declaration's presence** — a host with no `# fastlane-cardless:` line stays silent (its card-less prefixes can't be told from holes), so no version-skew noise for un-regenerated adopters.
- `tests/test_check_fastlane_symmetry.py` — +6 tests (reverse-silent-without-declaration, declared-card-less-green, undeclared-armed-warns, multi-declared-clears-all, both-legs-fire-together, `_declared_cardless_prefixes` parser unit).
- `dist/bootstrap.py` — rebuilt via `src/build_bootstrap.py` (engine code changed); byte-pin clean + idempotent across two rebuilds.

**Posture — identical to the forward leg:** advisory-only, never exit-affecting, deliberately off `STRICT_SUBCHECKS`. Both legs are pure host-self cross-checks over the host's OWN two workflow surfaces (no canonical pivot), so neither can false-wall a host that customized its fast-lane prefixes. No cli.py change needed — the check just returns more findings, and cli.py's existing "fast-lane-symmetry advisory warning(s)" emit block iterates every returned finding by kind.
