### 💡 Session idea (dedup-checked against docs/ideas/ + roadmap)

**`run_ab.py record` should validate value shapes, not just key
presence** — because `bench/results/` is append-only-immutable, a typo'd
row (verdict `"PSS"`, a date that contradicts the run_id prefix, an
m1 value that is neither number nor per-task map) can never be edited
once merged, only tombstoned by a correction row. The recorder is the
last gate before immutability, so it is exactly where cheap shape checks
(verdict ∈ {PASS, FAIL}, date == run_id prefix, m1_* numeric or
str→numeric map, tasks ⊆ TASKS) pay off forever. Small, self-contained,
ordinary lane (`run_ab.py` is not a pin path).
