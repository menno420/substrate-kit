## What shipped

R11 (#506) gave every `docs/recipes/` graduation a `> **applies-when:** <signature>` badge (`content:<marker>` / `path:<glob>` tokens) and `check_recipe_applies_when` warns when the badge is **missing, empty, or malformed** — a *well-formedness* lint. But a well-formed badge can still be **dishonest**: it can name a `content:` marker the recipe prose never mentions, or a `path:` glob whose literal skeleton the recipe never describes — the drift where a marker is renamed in the body but the badge still cites the old token. The future S17 discovery check will grep an adopter tree for these tokens to nudge toward the matching recipe, so a token that doesn't even appear in its **own** recipe's body is a false lead.

S8 adds the complementary **honesty** lint, `check_recipe_signature_honesty`:

- For each recipe under `docs/recipes/` (except README.md) carrying a well-formed badge, each token is cross-checked against the recipe body.
- `content:<marker>` — the marker substring must appear (case-insensitively) somewhere in the body → else one advisory.
- `path:<glob>` — the glob's literal (non-wildcard) fragments must each appear in the body (`*.json` requires `.json`; `src/checks/*.py` requires `src/checks/` and `.py`). A pure-wildcard glob (`*`) has no literal skeleton → honest by default (fail-open).
- The **badge line itself is excluded** from the searched body, so a token can never satisfy honesty via its own declaration (tested).
- **Only well-formed tokens** are honesty-checked — a missing/empty badge or a malformed token stays R11's job, so no double-noise.
- The badge grammar (`_RE_APPLIES_WHEN` / `_RE_TOKEN` / `_HEADER_LINES`) is **imported from `check_recipe_applies_when` as the single source of truth**, so the two lints can never drift on what a valid token is. The intra-package import is un-aliased (the dist flattens every module into one namespace and strips these imports; an aliased name would NameError in dist) — verified by running `python3 dist/bootstrap.py check --strict` with no NameError.

**Posture:** advisory-only, wired on the `posture="advisory"` seam in `cli.py` (NOT `_extra_check_findings`), input-gated + fail-open, **off** `STRICT_SUBCHECKS` — a drifted signature is a reconcile nudge, not a defect to fail an adopter on. `EXPECTED_STRICT_SUBCHECKS` stays 7 (no new top-level `check_*` in the strict set; a `test_not_in_strict_subchecks` asserts it). Both shipped recipes (`pinned-feed-contract.md`, `advisory-to-born-red-gate.md`) carry honest signatures, so the lint is **silent on the kit's own tree** (like R11) — a `test_shipped_recipes_are_honest` pins that.

**Dist:** the checker is in `MODULE_ORDER` after `check_recipe_applies_when.py` (whose regexes it imports); `dist/bootstrap.py` rebuilt via `python3 src/build_bootstrap.py` and byte-pinned (idempotent across two rebuilds).
