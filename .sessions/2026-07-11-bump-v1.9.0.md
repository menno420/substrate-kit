## 💡 Session idea

`build_release_json.py --verify-only` checks the version stamp, the
CHANGELOG section, and the dist header — but not the CHANGELOG's footer
tag-link line (`[X.Y.Z]: …/releases/tag/vX.Y.Z`). A cut that forgets the
link renders the new section header as plain unlinked text on GitHub and
the drift is invisible until a human scrolls the footer. One more refusal
line in verify (`section [X.Y.Z] present but footer link missing`) makes
the whole CHANGELOG cut machine-checked — enforce, don't exhort.
