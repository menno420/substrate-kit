### 💡 Session idea (dedup-checked against docs/ideas/ + roadmap)

**`adopt --lane <name>` — scaffold the per-lane shape, don't just check
it.** v1.4.0 makes the checker config-aware, but a second Project adopting
into a SHARED repo still hand-creates `control/status-<lane>.md`, hand-edits
`heartbeat_files`, and hand-splits the README wording. A `--lane` flag on
`adopt` (or a small `bootstrap lane add <name>` subcommand) would: plant
`control/status-<lane>.md` from `control-status.md.tmpl` (skip-if-exists),
append the lane to `heartbeat_files` via `save_config`, and leave
`inbox.md`/`README.md` single (skip-if-exists already does). That turns the
documented pattern into a one-command shape, the same enforce-don't-exhort
arc every kit convention follows (checker shipped first — this session;
scaffolding second). Anchors: `engine/adopt.py` `_PLANT_MAP`
(`control-status.md.tmpl` → parametrized dest), `Config.heartbeat_files`,
`cli.py` adopt arg parsing. Recorded in-card (single-command scope); groom
pass can file it.
