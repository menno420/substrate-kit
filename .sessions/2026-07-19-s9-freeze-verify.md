## What shipped

R10 (#505) made a `--json`/`--commit-results` run self-citing: `--freeze` writes a `<output>.freeze` sidecar carrying `sha256(blob)` over the exact output bytes plus a paste-ready reproduce command, so a committed `results.json` is tamper-evident *by hand* (`sha256sum <output>` verifies it). S9 adds the **automated inverse**, `--verify PATH` on `scripts/measure_grounded_skills.py`:

- Point it at a frozen artifact **or** its `<artifact>.freeze` sidecar — either resolves to the same `(artifact, sidecar)` pair (`_verify_frozen`).
- It re-hashes the artifact's exact on-disk bytes (the same bytes `--freeze` hashed and the same bytes `sha256sum <artifact>` sees) and compares to the digest the sidecar pinned.
- Exit **0** on match, **1** on a genuine hash mismatch (tamper / drift), **2** (via `ap.error`) on a missing artifact, missing sidecar, or a malformed / `sha256`-less sidecar — a *can't-verify* structural error kept distinct from a *verified-and-failed* mismatch (both non-zero, so the spec's "non-zero on mismatch" holds either way).
- **Standalone mode:** it short-circuits right after `parse_args`, *before* any target resolution, so it needs none of the `--clone`/`--local`/`--json` measurement flags. It is the scripted form of the `sha256sum <artifact>` line the freeze block already prints.

**Posture:** pure additive CLI mode on a standalone `scripts/` file — **not in `MODULE_ORDER`, so `dist/bootstrap.py` is untouched** (byte-identical, like R3/R9/R10). No engine seam, no strict-subcheck, no gate.

Files: `scripts/measure_grounded_skills.py` (`--verify` arg + post-parse short-circuit + `_verify_frozen()` helper + docstring), `tests/test_measure_grounded_skills.py` (+8 tests: `--freeze`→`--verify` round-trip pass, sidecar-path accept, tampered-artifact fail-1 with the sidecar left untouched, missing-artifact / missing-sidecar / malformed-sidecar / no-`sha256`-field usage-error-2, standalone-no-measure via a `measure_repo`-never-runs assertion).
