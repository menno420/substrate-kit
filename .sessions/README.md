# Session logs

Per-session logs live here as `<date>-<slug>.md`, newest first. Create the log as the session's FIRST commit with a born-red status (`> **Status:** `in-progress``) so in-flight work is visible to parallel sessions, then flip it to `complete` as the deliberate LAST step once the close-out is written — a half-done session never reads as finished. Before it counts as complete, a log must carry these markers: Status badge, Session idea, Previous-session review, Model line (`- **📊 Model:** <model> · <effort> · <task-class>` — session-close harvests it into `telemetry/model-usage.jsonl`, band KL-3).

A **routine-fired lab session** additionally carries the line `Run type: routine · lab` in its card's header block (the D3 flow signal, band KL-4 — see `docs/operations/lab-loop.md`); owner/manually-started sessions omit it.

**Auto-draft (band KL-5):** if the card is missing at session end, `session-close` / the Stop hook auto-draft one from evidence (files touched since the session-start anchor, git HEAD movement, the verify command); an in-progress card missing its close-out gets the drafted section appended. A draft is a starting point, not a close-out — verify the evidence, resolve every `[[fill:]]` slot, then flip the Status badge; unresolved slots (and the `drafted` status) keep the card counting incomplete, reported distinctly by the checker (drafted ≠ completed).
