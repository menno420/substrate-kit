# Session logs

Per-session logs live here as `<date>-<slug>.md`, newest first. Create the log as the session's FIRST commit with a born-red status (`> **Status:** `in-progress``) so in-flight work is visible to parallel sessions, then flip it to `complete` as the deliberate LAST step once the close-out is written — a half-done session never reads as finished. Before it counts as complete, a log must carry these markers: Status badge, Session idea, Previous-session review, Model line (`- **📊 Model:** <model> · <effort> · <task-class>` — session-close harvests it into `telemetry/model-usage.jsonl`, band KL-3). The `📊 Model:` model segment is the **family-level model name your own harness/environment reports this session** (e.g. `fable-5`, `opus-4.8`, `sonnet-5`) — the committed card's self-report is the attribution ground truth (fleet standing rule, inbox ORDER 012 / fm model matrix 2026-07). Never copy it from an external surface (schedule/Routines screens are evidenced to misattribute — websites #59), and never record an exact model ID — family-level names only, never an exact model-ID token (dated or not).

The effort segment: live sessions self-report a real tier (`low` · `medium` · `high`); `unrecorded` is the sanctioned **terminal** value reserved for retroactive repair of a card whose author never reported one — never backfill an invented tier, and never use `unrecorded` on your own live card (the lint stays silent on it; the harvest records it verbatim).

A **routine-fired lab session** additionally carries the line `Run type: routine · lab` in its card's header block (the D3 flow signal, band KL-4 — see `docs/operations/lab-loop.md`); owner/manually-started sessions omit it.

**Auto-draft (band KL-5):** if the card is missing at session end, `session-close` / the Stop hook auto-draft one from evidence (files touched since the session-start anchor, git HEAD movement, the verify command); an in-progress card missing its close-out gets the drafted section appended. A draft is a starting point, not a close-out — verify the evidence, resolve every `[[fill:]]` slot, then flip the Status badge; unresolved slots (and the `drafted` status) keep the card counting incomplete, reported distinctly by the checker (drafted ≠ completed).

**Guard recipes:** when a card records friction→guard material for a *later* session (a "friction → guard candidates" entry, a deferred ⚑ flag), carry a one-line **guard recipe** naming the code anchors — function + file + the test target — not just the symptom. A symptom-only entry costs the next session a re-derivation grep pass; a recipe lets it land the guard in minutes (the KL-0→KL-1 lesson: KL-1 had to re-derive `_vendor_bootstrap` / `_ref_mine_log` by grep from KL-0's symptom-only friction notes).
