## What shipped

The grounded-skills measurement harness (`scripts/measure_grounded_skills.py`) emits a machine-readable `results.json` payload тАФ one entry per repo, each carrying M1..M4 metrics тАФ but a later reader (a downstream consumer, a `--verify`'ing session, a paper trail on a frozen artifact) had **no way to see WHICH commit of each repo the numbers were computed against, and whether the clone was full or shallow.** The only shallow signal on the payload was `merged["shallow"]` тАФ a per-repo bool used by the R3 refuse-to-publish gate on `--json`/`--commit-results` (#492), collapsing UNKNOWN (git unavailable) into False; it never surfaced HEAD's SHA.

**S13 turns that into first-class self-describing provenance** тАФ purely additive over the existing payload shape:

- `scripts/measure_grounded_skills.py`:
  - New `_clone_provenance(repo_dir) -> tuple[str, str | None]` helper riding the S5-shared `_git_truth.make_runner` + `require_full_history` seam for the tri-state depth verdict (`_git_truth.FULL` / `_git_truth.SHALLOW` / `_git_truth.UNKNOWN`), plus `git rev-parse HEAD` for the SHA.
  - Two fields added to `RepoResult`: `clone_depth: str = _git_truth.UNKNOWN`, `git_sha: str | None = None`.
  - Called from `measure_repo` after the M4 pass; wired into `results_json` under two new keys per repo.
  - Module docstring extended with a short paragraph describing the S13 provenance surface.
- `tests/test_measure_grounded_skills.py` тАФ +8 tests: non-git tree reports UNKNOWN + None (`test_clone_provenance_non_git_tree_reports_unknown`); full git repo reports FULL + a 40-char lowercase-hex SHA (`test_clone_provenance_full_git_repo_reports_full_and_sha`); monkeypatched `_git_truth.is_shallow=True` reports SHALLOW (`test_clone_provenance_shallow_repo_reports_shallow`); `measure_repo` populates both fields (`test_measure_repo_records_clone_provenance`); non-git-tree defaults coherent with M4 null (`test_measure_repo_non_git_tree_provenance_defaults`); `results_json` payload carries the two fields (`test_results_json_carries_clone_provenance`); non-git-tree JSON defaults (`test_results_json_provenance_defaults_for_non_git_tree`); end-to-end via `main()` writes provenance on the artifact (`test_json_writes_provenance_end_to_end`).
