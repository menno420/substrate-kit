## What shipped

The R7 advisory `check_wall_ledger_agreement` (PR #498) cross-checks `docs/CAPABILITIES.md` for self-contradiction: a durable `## Walls` *correction* row ("X is NOT a wall") vs the newest `## Append log` verdict on that same capability (its `capability|wall` type token). But it keyed on a **hardcoded `_FAMILIES` tuple seeded with only merge/arm/flip** — a NEW capability contradiction (branch-deletion, tag-push, routine-creation…) went uncaught until someone hand-added a family. That gap was named in R7's own `💡` session idea as "the fuller form of the same lint."

S12 builds that fuller form — **purely additive** over the seeded loop:

- `src/engine/checks/check_wall_ledger_agreement.py`:
  - `_extract_bold_title(bullet)` — the noun-phrase heading a row = the inner text of its first `**bold**` span (non-greedy + DOTALL so a title that wrapped across ledger lines is captured whole; internal whitespace collapsed).
  - `_canonicalize_title(title) -> frozenset[str]` — lowercase, split on non-alphanumeric runs, drop `_TITLE_STOPWORDS` (English function words + ledger-ubiquitous nouns `wall`/`capability`/`agent`/`PR`/`session`…) and sub-3-char tokens. Word-order and filler-word invariant.
  - `_keys_match(a, b)` — two canonical keys name the same capability when the smaller non-empty keyword set is a **SUBSET** of the other. Subset (not mere intersection) keeps a single shared generic word from false-pairing unrelated capabilities.
  - New derived pass appended to `check_wall_ledger_agreement` after the seeded loop: builds `{key → (verdict, title)}` for every bold-titled append-log entry (newest-first), then for each `## Walls` *correction* row with a bold title, finds the newest append-log entry whose key subset-matches and fires one `Finding(kind=WALL_LEDGER_DISAGREE_KIND)` on verdict disagreement — naming the capability by its Walls title. A row a seeded family already owns (its keyword regex matches) is **skipped so it is never double-reported**; each derived capability fires at most once.
- `tests/test_check_wall_ledger_agreement.py` — +8 tests: derived-disagreement-fires, derived-agreement-silent, requires-bold-title-both-sides, unrelated-titles-no-false-pair, not-double-reported-with-seeded, fires-once-per-capability (newest wins), `_extract_bold_title` unit, `_canonicalize_title`/`_keys_match` unit.
- `dist/bootstrap.py` — rebuilt via `src/build_bootstrap.py` (engine code changed); byte-pin clean + idempotent across two rebuilds.
