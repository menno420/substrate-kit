## 💡 Session idea

The regression harness this slice built (extract a generated workflow's
`run: |` block → resolve the `${{ }}` expressions → execute under `bash
-e` in a scratch repo) is the first time the kit ever RAN its own
generated bash instead of string-asserting it. Cheap generalization: a
template shell-lint test that walks EVERY generated workflow
(`live_ci_workflow`, the enabler, `substrate-gate.yml` variants), extracts
each `run: |` block, and at minimum runs `bash -n` (syntax check) on it —
a template edit that breaks bash syntax currently ships silently to every
adopter and only reds in THEIR CI (the tail-1 class was a semantic bug,
but a syntax bug would be fleet-wide breakage on upgrade). One fixture,
every template, near-zero runtime.
