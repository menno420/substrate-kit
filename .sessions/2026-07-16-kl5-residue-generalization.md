## 💡 Session idea

Migrate `status_in_progress` from substring-scan to value-parse: mid-session, the sham fixture exposed that the badge line's *prose* decides in-progress state — a card whose Status VALUE reads `complete` but whose trailing parenthetical still says "*(auto-drafted by substrate-kit …)*" counts in-progress, because `status_in_progress` substring-matches `drafted` anywhere in the line ("auto-drafted" contains it). The module already solved this exact class for the drafted-state check (`_status_value_drafted` parses the backticked VALUE via `_STATUS_VALUE_RE`); extending value-parsing to `status_in_progress` (with a substring fallback for badge lines that don't backtick their value) would make the gate judge what the badge *says*, not what its commentary mentions — and the failure it prevents is nasty to debug: a visibly-complete card held red as "badge still says in-progress". Dedup: no hits for `status_in_progress`/badge-value in docs/ideas/ (grepped this session).
