## What shipped

`src/engine/guards.py` already pins the kit's whole enforcement surface — the SET of guard surfaces, not just each surface's steps — across four registries: `REGISTRY` (ci.yml `kit-quality` steps, split SETUP/MIRRORS/KIT_ONLY), `WORKFLOW_JOB_CENSUS` (every `.github/workflows/*.yml` job, classified gate/alias/automation), `STRICT_SUBCHECKS` (the `bootstrap check --strict` sub-check set), and `HOOK_CENSUS` (the Claude Code lifecycle hooks, classified advisory/orientation/enforcing). But those registries are ENFORCED only by the kit-only meta-test `tests/test_guard_surface_census.py` — which an adopter never runs — and are **invisible at `check` time**. S10 makes the census VISIBLE:

- New `src/engine/checks/check_surface_census.py` — `surface_census_note(target, config)` reads the authoritative `guards.py` accessors (`counts()`, `workflow_job_census()`, `hook_census()`, `STRICT_SUBCHECKS` + the kind constants) and returns ONE concise informational line describing the guard surface this bootstrap ships. Sourced live, so a registry change flows straight into the surfaced counts — no hardcoded numbers.
- `src/engine/cli.py` — imported and emitted on `cmd_check`'s full-lane NOTE seam, right beside `native_gate_note` / `required_unverified_note`.
- `src/build_bootstrap.py` — `checks/check_surface_census.py` added to `MODULE_ORDER` immediately after `guards.py` (so its bare accessor names resolve in the concatenated dist namespace); `dist/bootstrap.py` rebuilt + byte-pinned (idempotent across two rebuilds).

The line it prints (live, this tree): `surface census — the guard surface this bootstrap ships (src/engine/guards.py, pinned by tests/test_guard_surface_census.py): 18 ci.yml kit-quality step(s) (setup 3 · mirrors 5 · kit-only 10), 6 workflow job(s) (gate 1 · alias 2 · automation 3), 7 \`check --strict\` sub-check(s), 4 lifecycle hook(s) (advisory 3 · orientation 1 · enforcing 0).`

**Posture — a NOTE, the extreme fail-open form of "advisory":** emitted, never a `Finding`, never counted toward the exit code, NOT in `STRICT_SUBCHECKS`. Mirrors the `native_gate_note` / `required_unverified_note` "surface info in check output, never a finding" pattern rather than the posture="advisory" *finding* checkers of S4–S8.
