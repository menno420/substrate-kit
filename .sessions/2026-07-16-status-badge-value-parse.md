## 💡 Session idea

Make a valueless Status badge a grammar finding in the added-card lane: the value-parse work exposed that `> **Status:**` with NO parseable value passes `has_status_badge` (presence) but returns `None` from `_status_badge_value`, so `status_in_progress` is False and the card falls through to the full completeness check — a card can declare no state at all and be graded as if it claimed `complete`. The added-card lane already treats a MISSING badge as a grammar finding ("born-red exempts the badge's VALUE, never its presence"); a present-but-valueless badge should get the same finding, since a badge that declares nothing is exactly as unparseable as no badge. Cheap: `check_added_card` gains one `_status_badge_value(text) is None` branch beside `has_status_badge`. Dedup: no hits for "valueless"/"badge value"/`_status_badge_value` in docs/ideas/ or CHANGELOG [Unreleased] (grepped this session; the mechanism is new in this PR).
