## ЁЯТб Session idea

**Enabler-side arm guard for card-only PRs (defense in depth for the #40
class):** the gate-side HOLD shipped here closes the loophole, but the
auto-merge-enabler still ARMS a PR whose entire diff is one in-progress
card тАФ arming is harmless now only because the gate holds. A cheap second
wall: the enabler workflow skips arming (with a "born-red card-only тАФ
arm deferred to the synchronize re-arm" notice) when the PR diff is a
single added `<sessions_dir>/*.md` file, so a future gate regression can
never re-open the premature-merge window. Same guard-stacking logic as
the label-disarm pair in `docs/operations/auto-merge-guards.md`.
