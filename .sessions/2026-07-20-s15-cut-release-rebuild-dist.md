## What shipped

An opt-in `--rebuild-dist` flag on `scripts/cut_release.py` (repo release-cut tooling) that folds FOLLOWUP checklist **step 2** — "Dist regen + byte-pin: `python3 src/build_bootstrap.py`, then confirm the printed byte count equals `wc -c dist/bootstrap.py`" — into the `--write` cut. It is the one follow-up that is a deterministic, network-free text regeneration, so it mechanizes cleanly.

- `scripts/cut_release.py`:
  - `rebuild_dist(root) -> int`: runs the canonical `python3 src/build_bootstrap.py` (the exact command the runbook names) as a subprocess under `--root`, parses the builder's `wrote … (N bytes)` line, and **byte-pins** the reported count against `dist/bootstrap.py`'s actual on-disk size — the manual "confirm the count equals `wc -c`" step, mechanized. Refuses (`CutError` → exit 1, clear message) on: builder script missing, subprocess non-zero exit (carries the builder's stderr), unparseable output, and byte-pin mismatch. Network-free (the builder only reads `src/engine`, writes `dist/`).
  - `followup_checklist(version, rebuilt_bytes)`: renders the checklist with step 2's two faces — the manual instruction (`rebuilt_bytes is None`) or a folded-in "DONE — folded in via --rebuild-dist (N bytes, byte-pin verified)" confirmation. `FOLLOWUP_CHECKLIST` gained a `{dist_step}` slot the helper fills.
  - `run(..., rebuild=False)`: guarded param — refuses `--rebuild-dist` without `--write` (the byte-pin is meaningless on a dry-run that changes nothing). After the `--write` bump lands, calls `rebuild_dist`, prints the confirmation, and threads the byte count into the checklist.
  - `main()`: adds the `--rebuild-dist` argparse flag; the module docstring documents the flag and its **NEVER-does** boundary (no workflow dispatch / push / tag / commit / release.json).
- `tests/test_cut_release.py` (+9, class `TestRebuildDist` + one helper-pin update): rebuild regenerates the dist + byte-pins + flips step 2 to DONE; `--write` alone does not rebuild; `--rebuild-dist` without `--write` refused (both the argparse path and a direct `run()` call); missing builder refused; byte-pin mismatch refused; unparseable output refused; builder non-zero exit refused with stderr surfaced; the two-face checklist helper. Fixtures use a faithful fake `src/build_bootstrap.py` stand-in (resolves its dist path the same `parents[1]` way and prints the real byte line) so the actual subprocess mechanics are exercised **without ever mutating the real tree or cutting a release**. Updated `test_followup_checklist_keywords_pinned_to_runbook` to build via the new helper.
