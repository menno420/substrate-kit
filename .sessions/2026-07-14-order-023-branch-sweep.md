# 2026-07-14 — ORDER 023: kit-templated scheduled branch-sweep workflow (BUILD half)

> **Status:** `complete`

About to (opening declaration): execute inbox ORDER 023 (BUILD half) — ship a
kit-templated SCHEDULED sweep workflow (`branch-sweep.yml`, daily cron +
`workflow_dispatch` with a `dry_run` input) that enumerates merged+closed
PRs, deletes their same-repo head refs matching the agent-branch patterns
(`claude/*`, `codex/*`, `bot/*`), skips any ref that is the head of an OPEN
PR, never touches the default branch, and logs every deletion and skip
reason. Deliberately NOT built on `pull_request: closed` (GITHUB_TOKEN-driven
merges never trigger workflows — the recursion-guard trap the ORDER names).
Mirrors the auto-merge-enabler's exact staging/install mechanism: staged
always under `<state_dir>/ci/`, installed live by `adopt
--wire-enforcement`, kit-owned regen once it exists. Release + adopter regen
half = v1.17.0 wave, coordinator-sequenced (not this session).

Flagged decisions (decide-and-flag, PL-001): (1) added a diverged-tip
skip beyond the ORDER's literal spec — a swept ref whose tip is not a
spent PR head may carry unmerged work, and deletion is the one
irreversible act in scope, so it is skipped and logged instead;
(2) sweep default patterns exclude `claim/*` (deletion scope ≠ arming
scope); (3) cron chosen daily 03:17 UTC (ORDER offered daily or
6-hourly) — litter accrues slowly and adopters can set
`branch_sweep.cron`; (4) `GITHUB_TOKEN` only (no `ROUTINE_PAT`
preference like the enabler) — least privilege, attribution irrelevant
for deletions.

💡 Session idea: ship a `bootstrap.py sweep --dry-run` local subcommand
that renders the sweep plan (existing ∩ spent − open − default) via the
same rules against the live API from a session, so an owner or agent can
preview exactly what the next scheduled run will delete without
dispatching the workflow — the same read-only preview the enabler
preflight gives arming. Worth having because the first fleet-wide sweep
will delete ~500 refs in one shot and a zero-risk preview builds the
trust that run needs.
