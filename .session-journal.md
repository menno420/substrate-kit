## ⚡ Quick reference

- Full local gate (mirrors CI `kit-quality`):
  `python3.10 -m pytest tests/ -q` ·
  `python3.10 src/build_bootstrap.py && git diff --exit-code dist/` ·
  `python3.10 -m ruff check src/engine/` ·
  `python3.10 dist/bootstrap.py check --strict --require-session-log --session-log .sessions/<your-card>.md` ·
  `python3.10 scripts/check_program_law.py` ·
  `python3.10 scripts/check_idea_index.py`
- Everything runs via `python3.10` — the CI floor (never bare `python3`).
- Session shape: born-red card = FIRST commit → PR opens immediately → work →
  flip `> **Status:** \`complete\`` = LAST commit. Ready tokens:
  complete/done/ready/final/merged/shipped.

## Environment & boot runbook

- Pure stdlib engine; `pip install` needed only for dev tools (`pytest`, `ruff`).
- The dist is generated: edit `src/engine/`, then rebuild
  (`python3.10 src/build_bootstrap.py`) — never edit `dist/bootstrap.py` by hand.

## Recurring problems + fixes

- **Tag push → HTTP 403** through the session git proxy. Releases go via the
  `release.yml` `workflow_dispatch` run (creates the tag in-Actions); v1.0.0
  was cut this way. Never retry a hand tag-push. (Friction issue #15.)
- **Until 👤 P10 lands** (ruleset still requires the two legacy contexts, not
  `kit-quality`): prefer merging your session PR **by hand via MCP** after
  verifying CI green on the final head — the #7/#9 incidents are what arming
  auto-merge early can do here.
- **MCP-created PRs**: the auto-merge-enabler DOES fire on them when mergeable
  at open, and labels added in a second call race the enabler's snapshot
  (the #17 incident; the enabler now re-reads labels fresh before arming).
- **CHANGELOG inserts**: keep `[Unreleased]` grouped by type — one `### Added`,
  one `### Fixed`, in keep-a-changelog order. Two sessions independently
  inserted a mid-section `### Fixed` (PR #14, PR #17); a structure checker is
  backlogged (`docs/ideas/changelog-unreleased-structure-checker-2026-07-09.md`).

## Past mistakes to avoid

- Don't trust a merged-PR count or file state from memory/MCP cache — verify
  against live GitHub (`git fetch` + API); the run's docs audit caught a
  wrong upgrade-PR credit (#44 vs #46) this way.
- Don't touch an open owner-gated PR's hunks from a parallel branch
  (PR #17 owns the current-state In-flight + Stability KL-5 tail until it merges).
